Vinyl collection & record-request system — the complete reference.  inner-groove-vinyl.vercel.app

Everything is one app at four web addresses: guests browse and request records, bartenders run the queue, the owner manages the TV slideshow, and staff manage the collection. This guide covers all four, plus the automatic behaviors, TV setup, and how to make changes safely.

They're all the same app — just different addresses.

1For guests — browsing & requesting

Guests open the patron browser by scanning a table QR, or the QR that appears on the TV when the queue is open.

  • Browse the collection — search and filter, tap a record for details.
  • Request to play — tap Request to play, choose Which side? (Side A or Side B), and enter a first name. The request joins the live queue. (Requests only go through when the bartender has the queue ON.)
  • Random Groove — spin the reel for a random available record.
  • Wishlist — suggest records the bar doesn't have yet. Leaving an email and joining the newsletter is optional (off by default).

2For bartenders — running the queue

Scan to open
the bar queue
…/bar

Open the bar queue at …/bar — scan the QR behind the bar (no PIN needed).

  • The toggle — the big button reads QUEUE IS ON — tap to turn off or QUEUE IS OFF — tap to turn on. Turn it ON to accept requests; OFF to stop them.
  • On the deck — the record currently playing. Tap Done when it finishes.
  • Up next — the request queue. Tap Play to send a record to the deck, or Skip to drop a request.
  • Recently played — a running history; tap Clear to wipe it.

3For the owner — the TV slideshow

The TV shows a rotating slideshow (full-screen when the queue is off, behind the queue panel when it's on). Slides advance every 8 seconds.

  • Slides come from a Google Drive folder named "TV Show." Drop image slides in and they sync to the TV.
  • Sync runs automatically at 5pm on weekdays and noon on weekends.
  • To push slides immediately, open Admin → Slides → Sync now.
  • Slides play in filename order.

Automatic behaviors

  • Midnight reset — the queue clears and flips OFF every night (Eastern time, daylight-saving aware).
  • Scheduled slide sync — 5pm weekdays, noon weekends.
  • Realtime — every screen updates live; no refreshing needed.
  • Empty-queue QR — when the queue is on but empty, the TV shows a scan-to-request QR.
  • Stay awake — the Display keeps the screen from sleeping on its own.

TV setup (quick version)

One-time setup per screen — see the separate TV Display Setup guide for full detail.

  • Bar display — the Silk browser (it shows as "Internet" on the Firestick home). Open the Display bookmark, then scroll all the way down with the remote to go fullscreen.
  • Projector display — the Downloader app's browser. Open the Display bookmark, then press the hamburger button twice to hide the toolbar.
  • Set the Fire TV screensaver to Never so the screen doesn't sleep.

Making changes (technical)

  • The app is a single index.html (React via CDN, no build step), deployed through GitHub → Vercel.
  • Test first. There's a separate test site inner-groove-vinyl-test.vercel.app (repo melk724/inner-groove-vinyl-test) on the same database — prove changes there before promoting to production (melk724/inner-groove-vinyl).
  • Schema changes run as SQL in Supabase first, then deploy the updated index.html.
  • The slide sync is a Supabase Edge Function (sync-slides) using the Drive API and stored secrets.
  • Pin every CDN script to an exact version. Babel is pinned to 7.29.7; the Supabase tag should be pinned too, to avoid a silent breakage.
